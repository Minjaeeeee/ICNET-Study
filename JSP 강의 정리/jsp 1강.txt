1. 웹 프로그래밍이란?
(1) 웹 어플리케이션을 구현하는 행위이ㅏ
(2) 웹이란 1개 이상의 사이트가 연결되어있는 인터넷 서비스의 한가지 형태
(3) 인터넷이란, 1개 이상의 네트워크가 연결되어 있는 형태를 말한다.

2. 웹의 구성
(1) Http, FTP, SMTP, POP, DHCP
  - HTTP: 우리가 많이보는 하이퍼 텍스트, 웹에서 매일 사용, 정보가 보여짐
  - FTP: 파일을 주고 받는 것이다. 회사 서버에 내려받거나 올릴때 이 프로토콜을 사용해서 진행함
  - SMTP: 메일과 관련됨
  - POP: 메일과 관련됨, 메일을 받는 것
  - DHCP: 동적으로 바뀌는 프로토콜

(2) 프로토콜 , IP , DNS, PORT
  - 프로토콜: 네트워크 상에서 약속된 통신규약
  - IP: 네트워크 상에서 컴퓨터를 식별할 수 있는 주소
  - DNS: IP주소를 인간이 쉽게 외우도록 맵핑한(인간이 보기 쉽게) 문자열
  - Port: 해당 컴퓨터의 구동되고 있는 프로그램을 구현할 수 있는 번호


3. JAVA 웹
  - JAVA 플랫폼(J2SE, J2EE, J2ME)중에서 J2EE를 이용한 웹 프로그래밍
  - J2SE: 모바일 제작, 
  - 컴포넌트: 웹 어플리케이션을 구현하기 위한 구성요소 (JSP문서, 서블릿 문서, HTML문서를 담고 있는게 컨테이너)

4. 웹 프로그래밍 동작(반드시 중요함)
  - 웹서버: 클라이언트의 요청에 의해 정보를 제공해주는 서버
  - 웹 브라우저: 웹 서버에 정보를 요청하고 웹서버로부터 정보를 받는 매개체, 이때 HTTP 프로토콜을 사용한다
  - 브라우저 주소창에 타이핑 -> 웹서버로 요청이감 -> 로직을 수행하기 위해 웹 어플리케이션 서버로 응답(요청)을 보냄 -> 데이터베이스의 데이터값을 가져올때 DB에 접근 -> 서버에서는 적절하게 로직이 수행하고 웹서버에 전달
    그리고 웹서버가 사용자에게 응답을 보내서 웹이 운영된다
